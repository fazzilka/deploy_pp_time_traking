## Назначение проекта

Проект **Time Tracking** предназначен для учёта задач и времени, затраченного на их выполнение. Система позволяет пользователю создавать задачи, запускать и останавливать таймер, сохранять интервалы работы и автоматически вычислять суммарное время по каждой задаче.

Основная идея проекта заключается в том, чтобы пользователь мог не просто записывать список дел, а понимать, сколько времени реально уходит на выполнение конкретных задач, проектов и направлений работы. Для этого в приложении реализованы таймер, история временных интервалов, дедлайны, приоритеты, статусы выполнения и отчёты по затраченному времени.

В системе можно создавать задачи с названием, описанием, сроком выполнения, приоритетом и привязкой к проекту. Задачу можно запустить в работу, поставить на паузу, остановить таймер, отметить как выполненную или удалить. Для задач со сроком выполнения отображается время до дедлайна в удобном формате: дни, часы и минуты.

## Проекты и группировка задач

В приложении реализована сущность **«Проект»**, которая позволяет группировать задачи по направлениям. Например, пользователь может создать проекты «Учёба», «Разработка backend», «Клиентский сайт» или «Личные дела» и распределять задачи между ними.

Для каждого проекта отображается статистика: количество задач, активные задачи, выполненные задачи и суммарное затраченное время. Это помогает понимать, какие направления занимают больше всего времени и где сосредоточена основная нагрузка.

## Личные и командные рабочие пространства

Приложение поддерживает личные и командные рабочие пространства. Пользователь может работать в персональном workspace или создать организацию для командной работы.

В организации можно добавлять зарегистрированных участников по email, назначать им роли и совместно вести проекты, задачи и отчёты. Добавление участника создаёт запись membership в базе данных, поэтому организация появляется у добавленного пользователя после обновления списка рабочих пространств или следующего входа в систему.

В командной части приложения можно видеть участников, их роли, статус, количество проектов, задач и учтённое время. Это делает проект не только личным тайм-трекером, но и инструментом для совместной работы над задачами.

## Отчёты и аналитика

В проекте реализован раздел отчётов, где пользователь может анализировать свою работу. В отчётах отображаются общее затраченное время, количество задач с записанным временем, среднее время за день, лучший день, топ задач и распределение времени по проектам.

Благодаря этому приложение помогает не только фиксировать время, но и анализировать продуктивность, находить самые затратные задачи и понимать, какие проекты занимают больше всего внимания.

## Развёртывание и наблюдаемость

Проект развёрнут на сервере и доступен по адресу:

**https://time-tracking.online/**

Для проекта настроен production deploy, CI/CD, сбор метрик и отображение логов в Grafana.

Production deploy выполняется через `scripts/deploy.sh`. Скрипт сохраняет текущий commit, создаёт backup PostgreSQL перед миграциями, поднимает новую версию приложения и проверяет работоспособность основных сервисов: `postgres`, `backend`, `frontend`, backend `/health` и frontend page.

Если новая версия не проходит build, migrations, compose up или healthchecks, скрипт откатывает репозиторий на предыдущий commit, пересобирает и поднимает предыдущую рабочую версию. Автоматическое восстановление базы данных из backup по умолчанию не выполняется, инструкция восстановления описана в `docs/deploy/production-deploy.md`.

## Запуск серверной части проекта

Запуск выполняется из папки `server/`.

1. Перейти в папку серверной части проекта:

```bash
cd server
````

2. Запустить все сервисы одной командой:

```bash
make run
```

3. Посмотреть журнал работы серверной части проекта:

```bash
make logs
```

4. Остановить сервисы:

```bash
make down
```

## Документация и наблюдаемость после запуска

После запуска сервисов доступны:

* программный интерфейс серверной части и автоматическая документация: `http://localhost:8000/docs`;
* метрики для системы сбора метрик: `http://localhost:8000/metrics`;
* система сбора метрик Prometheus: `http://localhost:9090`;
* система визуализации Grafana: `http://localhost:3000`;
* журналы приложений доступны в Grafana через источник данных Loki и в терминале через команду:

```bash
cd server && make logs
```

По умолчанию для локальной Grafana используются данные:

```text
login: admin
password: admin
```

## Итог

Таким образом, **Time Tracking** — это веб-приложение для управления задачами, учёта времени, анализа продуктивности и командной работы. Проект объединяет таймер, задачи, проекты, отчёты, рабочие пространства, роли участников, CI/CD, production deploy, метрики и логи в единую систему.
