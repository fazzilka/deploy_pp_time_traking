# Описание предметной области

## 1. Назначение системы

Система предназначена для учёта задач и интервалов времени, затраченного на выполнение этих задач. Основная цель системы — обеспечить удобное измерение времени по каждой задаче и корректное вычисление суммарного времени как по отдельной задаче, так и по всем задачам в целом.

Система используется для:
- создания и ведения списка задач;
- запуска и остановки измерения времени по задаче;
- хранения истории интервалов времени по каждой задаче;
- получения суммарного времени по задаче;
- получения сводной статистики по всем задачам и определения задач, на которые потрачено больше всего времени.

## 2. Основные сущности предметной области

### 2.1. Задача

**Задача** — это единица учёта работы, для которой измеряется затраченное время.

Задача содержит:
- идентификатор — уникальный номер задачи в системе;
- название — краткое наименование задачи;
- описание — развернутое описание задачи;
- список интервалов времени — последовательность периодов, когда задача выполнялась;
- суммарное время — накопленная длительность выполнения задачи, рассчитанная на основе интервалов времени.

Суммарное время хранится в виде числового значения, отражающего длительность. Рекомендуемый формат хранения — количество секунд, чтобы упростить вычисления и исключить неоднозначности.

### 2.2. Интервал времени

**Интервал времени** — это непрерывный период, в течение которого выполнялась задача.

Интервал времени содержит:
- время начала — дата и время запуска измерения;
- время окончания — дата и время остановки измерения.

Интервал времени может быть:
- **закрытым** — указаны и время начала, и время окончания;
- **активным** — указано только время начала, а время окончания отсутствует до остановки измерения.

Длительность интервала времени рассчитывается как разница между временем окончания и временем начала.

## 3. Правила и ограничения предметной области

### 3.1. Правила ведения интервалов времени

1. У одной задачи может быть много интервалов времени.
2. У одной задачи одновременно может существовать не более одного активного интервала времени.
3. При запуске измерения времени:
   - если активного интервала времени нет, создаётся новый активный интервал времени с заполненным временем начала;
   - если активный интервал времени уже существует, запуск измерения не должен создавать второй активный интервал времени.
4. При остановке измерения времени:
   - если активный интервал времени существует, у него заполняется время окончания, после чего он становится закрытым;
   - если активного интервала времени нет, остановка измерения не должна изменять данные задачи.

### 3.2. Правила вычисления суммарного времени

1. Суммарное время задачи равно сумме длительностей всех закрытых интервалов времени.
2. Активный интервал времени не должен увеличивать суммарное время до момента его закрытия.
3. При изменении списка интервалов времени суммарное время должно быть пересчитано так, чтобы отражать фактическую сумму длительностей.

### 3.3. Правила удаления

При удалении задачи должны быть удалены все связанные с ней интервалы времени, чтобы в системе не оставались данные без владельца.

## 4. Поиск и фильтрация

### 4.1. Поиск по названию

Система предоставляет возможность поиска задач по названию. Поиск выполняется по строке, введённой пользователем, и должен возвращать задачи, чьё название содержит указанную строку.

### 4.2. Фильтр по наличию времени больше нуля

Система предоставляет возможность отфильтровать задачи по признаку накопленного времени:
- при включённом фильтре отображаются только задачи, у которых суммарное время больше нуля;
- при выключённом фильтре отображаются все задачи.

## 5. Сводка и статистика

Система предоставляет сводку по времени, включающую:
- суммарное время по всем задачам — сумма суммарного времени каждой задачи;
- перечень задач с наибольшим временем — список задач, отсортированный по убыванию суммарного времени, с выводом первых задач из списка.

Сводка предназначена для анализа распределения времени и выявления наиболее трудоёмких задач.

## 6. Типовые сценарии использования

### 6.1. Создание задачи

Пользователь создаёт задачу, указывает название и при необходимости описание. После создания задача появляется в списке задач с нулевым суммарным временем.

### 6.2. Учёт времени по задаче

Пользователь выбирает задачу и запускает измерение времени. Система создаёт активный интервал времени. После окончания работы пользователь останавливает измерение времени. Система закрывает интервал времени и пересчитывает суммарное время задачи.

### 6.3. Просмотр карточки задачи

Пользователь открывает карточку задачи и видит:
- название и описание;
- суммарное время;
- список интервалов времени с датой и временем начала и окончания.

### 6.4. Получение сводки по времени

Пользователь открывает экран сводки времени и видит:
- суммарное время по всем задачам;
- список задач с наибольшим временем.

## 7. Термины

- **Задача** — объект учёта работы, для которого измеряется время.
- **Интервал времени** — период выполнения задачи между запуском и остановкой измерения.
- **Активный интервал времени** — интервал времени без времени окончания.
- **Закрытый интервал времени** — интервал времени с указанными временем начала и временем окончания.
- **Суммарное время** — накопленная длительность выполнения задачи, вычисленная по закрытым интервалам времени.
