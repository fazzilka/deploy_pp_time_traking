# Модель данных

## 1. Назначение

Документ описывает модель данных системы учёта задач и интервалов времени. Модель данных предназначена для хранения задач, истории интервалов времени выполнения и вычисленных суммарных значений времени.

## 2. Общие правила хранения времени

1. Время начала и время окончания интервала времени хранятся в формате дата и время в часовом поясе UTC.
2. Суммарное время задачи хранится в секундах как целое число.
3. Длительность интервала времени вычисляется как разница между временем окончания и временем начала в секундах.
4. Активный интервал времени не учитывается в суммарном времени до момента его закрытия.

## 3. Сущности и атрибуты

### 3.1. Задача

Сущность "Задача" хранит данные о задаче и суммарное время выполнения.

**Атрибуты:**
- `id` — идентификатор задачи. Тип: целое число. Уникальный.
- `title` — название задачи. Тип: строка. Обязательный.
- `description` — описание задачи. Тип: строка. Необязательный.
- `total_time_seconds` — суммарное время задачи в секундах. Тип: целое число. Неотрицательный.
- `deadline` — срок выполнения задачи. Тип: дата. Может отсутствовать.
- `priority` — приоритет задачи. Тип: строка. Допустимые значения: `lowest`, `low`, `medium`, `high`, `highest`. Значение по умолчанию: `medium`.
- `created_at` — дата и время создания задачи в UTC. Тип: дата и время.
- `updated_at` — дата и время последнего изменения задачи в UTC. Тип: дата и время.

**Примечания:**
- Значение `total_time_seconds` должно быть равно сумме длительностей всех закрытых интервалов времени, связанных с задачей.
- `deadline` не влияет на таймер и используется для планирования/отображения.
- `priority` обязателен для новых и существующих задач; при миграции старые задачи получают `medium`.

### 3.2. Интервал времени

Сущность "Интервал времени" хранит данные о периоде выполнения задачи.

**Атрибуты:**
- `id` — идентификатор интервала времени. Тип: целое число. Уникальный.
- `task_id` — идентификатор задачи, к которой относится интервал времени. Тип: целое число. Обязательный.
- `started_at` — время начала интервала времени в UTC. Тип: дата и время. Обязательный.
- `ended_at` — время окончания интервала времени в UTC. Тип: дата и время. Может отсутствовать, если интервал активный.
- `created_at` — дата и время создания записи интервала времени в UTC. Тип: дата и время.

**Состояния интервала времени:**
- закрытый интервал времени: `ended_at` задано;
- активный интервал времени: `ended_at` отсутствует.

**Ограничения:**
- время окончания не может быть меньше времени начала;
- для одной задачи одновременно может существовать не более одного активного интервала времени.

## 4. Связи между сущностями

1. Одна задача может иметь много интервалов времени.
2. Один интервал времени относится ровно к одной задаче.

Связь: `Task (1) -> (N) TimeInterval`

## 5. Ограничения целостности

### 5.1. Уникальность активного интервала времени

Для одной задачи должен существовать не более одного активного интервала времени.

Реализация на уровне базы данных должна обеспечивать невозможность одновременного существования двух интервалов времени с одинаковым `task_id` и отсутствующим `ended_at`.

### 5.2. Ссылочная целостность

Поле `task_id` в интервале времени должно ссылаться на существующую задачу.

При удалении задачи должны удаляться все связанные интервалы времени.

### 5.3. Неотрицательное суммарное время

Поле `total_time_seconds` у задачи должно быть неотрицательным.

## 6. Представление данных в программном интерфейсе

### 6.1. Представление задачи

Задача в программном интерфейсе приложения должна возвращаться с основными полями и списком интервалов времени.

Рекомендуемые поля:
- идентификатор задачи;
- название;
- описание;
- суммарное время;
- срок выполнения;
- приоритет;
- список интервалов времени.

### 6.2. Представление интервала времени

Интервал времени в программном интерфейсе приложения должен включать:
- идентификатор интервала времени;
- время начала;
- время окончания (если интервал закрыт).

## 7. Пример вычисления суммарного времени

Если у задачи есть два закрытых интервала времени:
- интервал времени 1: 600 секунд
- интервал времени 2: 900 секунд

Тогда суммарное время задачи равно 1500 секунд.

Если у задачи также есть активный интервал времени, он не должен изменять суммарное время до момента остановки измерения времени.
