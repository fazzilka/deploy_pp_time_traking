# Интерфейс пользователя

## 1. Назначение

Пользовательский интерфейс предназначен для управления задачами, учёта интервалов времени выполнения задач, а также просмотра сводной информации по времени. Интерфейс включает набор экранов, каждый из которых поддерживает отдельный сценарий работы пользователя.

## 2. Общие принципы интерфейса

1. Пользовательский интерфейс должен предоставлять доступ ко всем операциям: создание, чтение, обновление и удаление задач, управление таймером и просмотр сводки времени.
2. Суммарное время задачи должно отображаться в человекочитаемом виде (часы, минуты, секунды), при этом хранение и вычисления выполняются в секундах.
3. Управление таймером должно быть доступно на экране списка задач.
4. Интерфейс должен предотвращать действия, нарушающие правила предметной области, либо показывать понятное сообщение об ошибке.
5. Пользователь может работать в личном или командном workspace. Текущий workspace выбирается в верхней навигации и влияет на задачи, проекты и отчёты.
6. В навигации, профиле и командной таблице пользователь отображается через автоматически сгенерированную SVG-аватарку. Если `avatar_seed` недоступен, интерфейс использует fallback из email, username, id или `avatar_letter`.

## 3. Экраны и сценарии

### 3.1. Экран списка задач с управлением таймером

#### Назначение
Экран отображает список всех задач и позволяет выполнять основные действия без перехода в карточку задачи: запуск и остановку измерения времени, переход в карточку задачи, создание новой задачи, поиск и фильтрацию.

#### Отображаемые данные
- название задачи;
- краткое описание (если есть);
- проект или метка “Без проекта”;
- суммарное время задачи;
- признак активного интервала времени (например, статус “таймер запущен”).

#### Элементы управления
- кнопка “Создать задачу”;
- строка поиска по названию;
- переключатель фильтра “Суммарное время больше нуля”;
- фильтр по проекту: “Все проекты”, “Без проекта” или конкретный проект;
- кнопка “Запустить таймер” для каждой задачи;
- кнопка “Остановить таймер” для каждой задачи (доступна, если таймер запущен);
- переход в карточку задачи по нажатию на строку задачи.

#### Сценарии
1. Пользователь открывает список задач и видит задачи с их суммарным временем.
2. Пользователь вводит текст в поле поиска и получает отфильтрованный список.
3. Пользователь включает фильтр “Суммарное время больше нуля” и видит только задачи с накопленным временем.
4. Пользователь выбирает проект и видит только задачи выбранного проекта.
5. Пользователь нажимает “Запустить таймер” у задачи:
   - если активного интервала времени нет, таймер запускается и задача помечается как активная;
   - если активный интервал времени уже существует, интерфейс показывает сообщение об ошибке.
6. Пользователь нажимает “Остановить таймер”:
   - если активный интервал времени есть, он закрывается и суммарное время обновляется;
   - если активного интервала времени нет, интерфейс показывает сообщение об ошибке.

---

### 3.2. Экран создания задачи

#### Назначение
Экран предназначен для создания новой задачи.

#### Поля ввода
- название (обязательное);
- описание (необязательное).
- проект (необязательное поле, по умолчанию “Без проекта” или выбранный фильтр проекта).

#### Элементы управления
- кнопка “Сохранить”;
- кнопка “Отмена”.

#### Сценарий
1. Пользователь открывает экран создания задачи.
2. Пользователь вводит название и при необходимости описание.
3. Пользователь нажимает “Сохранить”.
4. Система создаёт задачу и переводит пользователя на экран списка задач или в карточку созданной задачи.

---

### 3.3. Экран карточки задачи

#### Назначение
Экран предназначен для просмотра подробной информации о задаче и истории интервалов времени.

#### Отображаемые данные
- идентификатор задачи;
- название;
- описание;
- проект;
- суммарное время задачи;
- список интервалов времени.

Для каждого интервала времени отображается:
- время начала;
- время окончания (если интервал закрыт);
- длительность интервала времени (если интервал закрыт);
- признак активного интервала времени (если время окончания отсутствует).

#### Элементы управления
- кнопка “Редактировать задачу”;
- выбор проекта для переноса задачи;
- кнопка “Удалить задачу”;
- кнопка “Назад к списку задач”.

#### Сценарии
1. Пользователь открывает карточку задачи из списка задач.
2. Пользователь просматривает суммарное время и историю интервалов времени.
3. Пользователь переходит к редактированию задачи.
4. Пользователь удаляет задачу и возвращается к списку задач.

---

### 3.4. Экран редактирования задачи

#### Назначение
Экран предназначен для изменения названия и описания существующей задачи.

#### Поля ввода
- название (обязательное);
- описание (необязательное).

#### Элементы управления
- кнопка “Сохранить”;
- кнопка “Отмена”.

#### Сценарий
1. Пользователь открывает экран редактирования из карточки задачи.
2. Пользователь изменяет название и (или) описание.
3. Пользователь нажимает “Сохранить”.
4. Система обновляет задачу и возвращает пользователя в карточку задачи.

---

### 3.5. Экран сводки времени

#### Назначение
Экран предназначен для просмотра агрегированной информации по времени выполнения задач.

#### Отображаемые данные
- суммарное время по всем задачам;
- перечень задач с наибольшим временем.
- распределение времени по проектам;
- перечень проектов с наибольшим временем.

Для каждой задачи в перечне отображается:
- название;
- суммарное время.

#### Элементы управления
- выбор количества задач в перечне (необязательное поле, по умолчанию 10);
- переход в карточку задачи при выборе задачи из перечня;
- кнопка “Назад к списку задач”.

#### Сценарии
1. Пользователь открывает экран сводки времени.
2. Пользователь видит суммарное время по всем задачам.
3. Пользователь видит перечень задач с наибольшим временем и может перейти в карточку выбранной задачи.
4. Пользователь видит, какие проекты занимают больше всего времени.

---

### 3.6. Экран проектов

#### Назначение
Экран отображает проекты пользователя и позволяет создать новый проект.

#### Отображаемые данные
- название проекта;
- описание;
- цвет проекта;
- количество задач;
- количество активных задач;
- суммарное время проекта;
- прогресс относительно самого длительного проекта;
- виртуальная карточка “Без проекта”, если есть задачи без проекта.

#### Элементы управления
- кнопка “Создать проект”;
- карточка проекта для перехода на страницу проекта.

#### Сценарии
1. Пользователь открывает экран “Проекты”.
2. Пользователь видит активные проекты и их суммарное время.
3. Пользователь создаёт проект, указав название, описание и цвет.
4. Пользователь открывает карточку проекта.

---

### 3.7. Страница проекта

#### Назначение
Страница проекта показывает задачи и статистику конкретного проекта.

#### Отображаемые данные
- название и описание проекта;
- суммарное время;
- количество задач;
- количество активных задач;
- количество задач с временем;
- список задач проекта;
- топ задач проекта по времени.

#### Элементы управления
- кнопка “Редактировать”;
- кнопка “Архивировать”;
- кнопка “Создать задачу”;
- поиск задач проекта;
- фильтр задач с временем;
- кнопки запуска, остановки и удаления задачи.

#### Сценарии
1. Пользователь открывает проект.
2. Пользователь создаёт задачу внутри проекта.
3. Пользователь запускает и останавливает таймер задачи проекта.
4. Пользователь редактирует название, описание или цвет проекта.
5. Пользователь архивирует проект; задачи и история времени сохраняются.

---

### 3.8. Экран команды

#### Назначение
Экран "Команда" предназначен для управления участниками текущего workspace, их ролями и обзором командной статистики.

#### Отображаемые данные
- название текущего workspace;
- количество участников;
- количество проектов и задач workspace;
- суммарное время по workspace;
- таблица участников с SVG-аватаркой, email, ролью, статусом, количеством задач и временем;
- описание ролей `Owner`, `Team Lead`, `Member`, `Viewer`;
- блок добавления участников по email.

#### Элементы управления
- переключатель workspace в верхней навигации;
- пункт “Создать организацию” в переключателе workspace;
- ссылка навигации “Команда”;
- кнопка “Пригласить участника”;
- кнопка “Настройки команды”;
- форма добавления участника по email;
- выбор роли при добавлении участника;
- форма изменения названия и описания workspace;
- кнопка “Сгенерировать новую аватарку” на странице профиля пользователя;
- действия изменения роли и удаления участника, если у текущего пользователя достаточно прав.

#### Сценарии
1. Пользователь создаёт организацию через workspace switcher и становится её Owner.
2. Пользователь открывает вкладку “Команда” и видит участников текущего workspace.
3. Owner или Team Lead вводит email уже зарегистрированного пользователя и добавляет его в workspace.
4. Добавленный пользователь после обновления страницы или следующего входа видит организацию в workspace switcher.
5. Если пользователь не зарегистрирован, интерфейс показывает ошибку “Пользователь с таким email не найден”.
6. Если пользователь уже состоит в workspace, интерфейс показывает ошибку “Пользователь уже состоит в команде”.
7. Owner может открыть “Настройки команды” и изменить название или описание workspace.
8. Owner может менять роли участников и удалять участников, кроме единственного владельца workspace.
9. Team Lead может добавлять участников, но не назначает Owner и не повышает до Team Lead.
10. Member может создавать задачи и запускать таймер, но не создаёт проекты.
11. Viewer видит командные данные, но не может изменять workspace, задачи, проекты или запускать таймер.

## 4. Обработка ошибок и уведомления

1. Если при запуске таймера у задачи уже есть активный интервал времени, интерфейс должен показать сообщение о невозможности запуска таймера.
2. Если при остановке таймера у задачи нет активного интервала времени, интерфейс должен показать сообщение о невозможности остановки таймера.
3. При ошибках сети или сервера интерфейс должен показать сообщение о том, что операция не выполнена, и предложить повторить попытку.

## 5. Навигация

1. Основной входной экран — экран списка задач.
2. Из списка задач пользователь может перейти:
   - на экран создания задачи;
   - в карточку задачи;
   - на экран сводки времени.
3. Из основного меню пользователь может перейти на экран проектов.
4. Из карточки задачи пользователь может перейти:
   - на экран редактирования задачи;
   - обратно на экран списка задач.
5. Верхняя навигация содержит переключатель workspace и разделы “Таймер”, “Проекты”, “Отчёты”, “Команда”, “Профиль”.
6. Переключатель workspace показывает личное пространство, организации и действие “Создать организацию”.
7. При переключении workspace список задач, проекты и отчёты обновляются в контексте выбранного workspace.
