# План тестирования

## 1. Назначение

План тестирования описывает сценарии проверки функциональности системы учёта задач и интервалов времени. Цель тестирования — подтвердить корректность работы операций управления задачами, работы таймера и формирования сводной информации.

## 2. Область тестирования

Тестирование включает:
- операции управления задачами;
- операции запуска и остановки измерения времени;
- поиск и фильтрацию задач;
- формирование сводной информации;
- обработку ошибочных сценариев.

## 3. Предварительные условия

Перед выполнением сценариев должны быть выполнены условия:
1. Приложение запущено.
2. База данных доступна.
3. Доступны маршруты программного интерфейса приложения.

## 4. Сценарии тестирования

### Сценарий 1. Создание задачи
**Шаги:**
1. Выполнить запрос создания задачи с названием и описанием.
2. Получить задачу по идентификатору из ответа.

**Ожидаемый результат:**
- задача создана;
- суммарное время равно нулю;
- список интервалов времени пуст.

---

### Сценарий 2. Изменение задачи
**Шаги:**
1. Создать задачу.
2. Выполнить запрос изменения названия и описания.
3. Получить задачу по идентификатору.

**Ожидаемый результат:**
- название и описание обновлены;
- суммарное время и интервалы времени не изменились.

---

### Сценарий 3. Удаление задачи и удаление связанных интервалов времени
**Шаги:**
1. Создать задачу.
2. Запустить измерение времени и остановить его, чтобы появился закрытый интервал времени.
3. Удалить задачу.
4. Попытаться получить задачу по идентификатору.

**Ожидаемый результат:**
- задача удалена;
- интервалы времени, связанные с задачей, удалены;
- получение по идентификатору возвращает ошибку отсутствия сущности.

---

### Сценарий 4. Запуск измерения времени при отсутствии активного интервала времени
**Шаги:**
1. Создать задачу.
2. Выполнить запуск измерения времени.

**Ожидаемый результат:**
- создан активный интервал времени;
- время окончания отсутствует.

---

### Сценарий 5. Повторный запуск измерения времени при наличии активного интервала времени
**Шаги:**
1. Создать задачу.
2. Выполнить запуск измерения времени.
3. Повторно выполнить запуск измерения времени.

**Ожидаемый результат:**
- второй запуск отклонён;
- возвращается ошибка конфликта состояния;
- активный интервал времени остаётся единственным.

---

### Сценарий 6. Остановка измерения времени при наличии активного интервала времени
**Шаги:**
1. Создать задачу.
2. Запустить измерение времени.
3. Остановить измерение времени.
4. Получить задачу по идентификатору.

**Ожидаемый результат:**
- активный интервал времени закрыт;
- время окончания заполнено;
- суммарное время увеличилось на длительность закрытого интервала времени.

---

### Сценарий 7. Остановка измерения времени при отсутствии активного интервала времени
**Шаги:**
1. Создать задачу.
2. Выполнить остановку измерения времени без запуска.

**Ожидаемый результат:**
- операция отклонена;
- возвращается ошибка конфликта состояния.

---

### Сценарий 8. Поиск и фильтрация задач
**Шаги:**
1. Создать две задачи с разными названиями.
2. Выполнить поиск по части названия первой задачи.
3. Запустить и остановить измерение времени по одной задаче, чтобы суммарное время стало больше нуля.
4. Выполнить фильтрацию по наличию суммарного времени больше нуля.

**Ожидаемый результат:**
- поиск возвращает только задачи, соответствующие строке поиска;
- фильтр возвращает только задачи с суммарным временем больше нуля.

---

### Сценарий 9. Формирование сводной информации
**Шаги:**
1. Создать несколько задач.
2. Для части задач создать закрытые интервалы времени.
3. Запросить сводную информацию.

**Ожидаемый результат:**
- возвращается суммарное время по всем задачам;
- возвращается перечень задач с наибольшим временем, отсортированный по убыванию времени.
