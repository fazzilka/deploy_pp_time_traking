# Руководство по запуску

## 1. Назначение

Руководство описывает порядок запуска серверной части приложения, а также способы проверки работоспособности, просмотра логов и метрик мониторинга.

## 2. Предварительные требования

Для запуска должны быть установлены:
- система контроля версий Git;
- среда контейнеризации Docker;
- инструмент управления контейнерами Docker Compose;
- утилита make.

## 3. Структура проекта

В корне проекта находятся каталоги:
- `server` — серверная часть приложения;
- `Frontend` — клиентская часть приложения;
- `docs` — документация;
- `samples` — примеры данных и запросов.

## 4. Запуск приложения

1. Перейти в каталог `server`:
```bash
cd server
```
Запустить приложение:
```bash
make run
```
Убедиться в запуске:

- выполнить запрос проверки работоспособности:
```
GET /health
```
5. Остановка приложения

Для остановки выполнить:
```bash
cd server
make down
```
6. Просмотр логов

Для просмотра логов серверной части выполнить:
```bash
cd server
make logs
```
7. Проверка программного интерфейса приложения

Примеры запросов находятся в файле:

- samples/example_requests.http

Запросы можно выполнять любым клиентом, который поддерживает отправку запросов к программному интерфейсу приложения.

8. Мониторинг и просмотр логов

После запуска доступны компоненты наблюдаемости:

 1. Метрики приложения доступны по маршруту:
```
GET /metrics
```
 2. Визуализация метрик и просмотр логов доступны через Grafana.
  - Адрес Grafana указывается в настройках контейнеров и зависит от выбранных портов.
 3. Сбор метрик выполняется Prometheus.
  - Источником метрик является маршрут /metrics серверной части.
 4. Сбор и хранение логов выполняется связкой Promtail и Loki.
  - Promtail читает логи контейнеров и отправляет их в Loki.
  - Loki хранит логи и предоставляет доступ для Grafana.

9. Типовые команды обслуживания
- Проверка форматирования и статического анализа:
```bash
cd server
make lint
```

Проверка типов:
```bash
cd server
make type
```
Установка проверок перед коммитом:
```bash
cd server
make hooks
```
Применение миграций базы данных:
```bash
cd server
make upgrade
```
