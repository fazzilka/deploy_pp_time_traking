# Отчет о результатах

## 1. Назначение

Отчет фиксирует итоговое состояние проекта после выполнения работ по реализации серверной части и подготовке документации.

## 2. Реализованный функционал

Реализованы возможности:
- создание задач с названием и описанием;
- завершение задачи без удаления и возврат задачи в работу;
- создание личных и командных рабочих пространств;
- добавление зарегистрированных участников в workspace по email;
- появление добавленного workspace у участника после обновления списка workspace;
- управление ролями участников workspace;
- совместные проекты, задачи и отчёты в рамках workspace;
- получение списка задач с поиском по названию и фильтром по суммарному времени больше нуля;
- получение задачи по идентификатору со списком интервалов времени;
- изменение названия и описания задачи;
- удаление задачи с удалением связанных интервалов времени;
- запуск измерения времени по задаче с созданием активного интервала времени;
- остановка измерения времени по задаче с закрытием активного интервала времени и пересчётом суммарного времени;
- получение сводной информации по времени, включая суммарное время по всем задачам и перечень задач с наибольшим временем;
- получение командной сводки по workspace и участникам;
- настройка названия и описания workspace владельцем;
- автоматическая генерация уникальных SVG-аватарок пользователей по `avatar_seed` без хранения файлов изображений;
- проверка работоспособности приложения;
- получение метрик для мониторинга.

## 3. Соответствие требованиям

1. Функциональные требования выполнены:
- реализованы операции управления задачами;
- реализованы операции управления таймером;
- реализованы поиск, фильтрация и сводная информация.
- реализованы рабочие пространства, участники и роли для командной работы;
- реализовано разделение прав: проекты создают Owner и Team Lead, задачи создают Owner, Team Lead и Member, Viewer работает в режиме чтения.

2. Нефункциональные требования выполнены в части:
- предусмотрено логирование и сбор метрик;
- обеспечена возможность запуска в контейнерах;
- предусмотрены команды make для типовых операций запуска и обслуживания.

## 4. Документация и примеры

Подготовлены документы:

- требования;
- модель данных;

Подготовлены примеры:

## 5. Результаты тестирования

Сценарии тестирования выполнены. Выявленные дефекты исправлены. Система готова к демонстрации.

## 6. Ограничения

Приглашения в команду реализованы в базовом варианте: владелец или тимлид вводит email уже зарегистрированного пользователя, после чего backend создаёт запись `workspace_members`, и пользователь получает доступ к workspace после обновления списка workspace или следующего входа. Ссылки-приглашения, pending invitations, accept invite page, email-рассылка и push-уведомления вынесены в следующий этап.
