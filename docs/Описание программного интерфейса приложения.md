## Правило работы с интервалами времени

Для одной задачи одновременно может существовать только один активный интервал времени.
Если для задачи уже есть интервал времени без времени окончания, повторный запуск измерения времени возвращает ошибку с кодом `409`.
Активный интервал времени не увеличивает суммарное время задачи до момента остановки.

## Маршруты приложения

### Получение списка задач

`GET /api/v1/tasks`

Параметры запроса:
- `search` — поиск по названию задачи.
- `has_time=true` — показать только задачи, у которых суммарное время больше нуля.

Пример ответа:

```json
[
  {
    "id": 1,
    "title": "Подготовить отчёт",
    "description": "Собрать данные и оформить документ",
    "total_time_seconds": 5400,
    "intervals": [
      {
        "id": 3,
        "started_at": "2026-02-23T10:00:00Z",
        "finished_at": "2026-02-23T11:30:00Z"
      }
    ]
  }
]
```

### Получение задачи по идентификатору

`GET /api/v1/tasks/{task_id}`

### Создание задачи

`POST /api/v1/tasks`

Пример запроса:

```json
{
  "title": "Изучить требования",
  "description": "Подготовить список задач"
}
```

Пример ответа:

```json
{
  "id": 2,
  "title": "Изучить требования",
  "description": "Подготовить список задач",
  "total_time_seconds": 0,
  "intervals": []
}
```

### Изменение задачи

`PATCH /api/v1/tasks/{task_id}`

### Удаление задачи

`DELETE /api/v1/tasks/{task_id}`

При удалении задачи удаляются все связанные интервалы времени.

### Запуск измерения времени

`POST /api/v1/tasks/{task_id}/timer/start`

Если активный интервал времени уже существует, возвращается ошибка с кодом `409`.

### Остановка измерения времени

`POST /api/v1/tasks/{task_id}/timer/stop`

Если активный интервал времени отсутствует, возвращается ошибка с кодом `409`.
После остановки измерения времени длительность добавляется в `total_time_seconds` в секундах.

### Получение сводки

`GET /api/v1/summary`

Параметры запроса:
- `limit` — ограничение количества задач в списке задач с наибольшим временем. По умолчанию значение равно `10`.

Пример ответа:

```json
{
  "total_time_seconds": 7200,
  "top_tasks": [
    {
      "id": 1,
      "title": "Подготовить отчёт",
      "description": "Собрать данные и оформить документ",
      "total_time_seconds": 5400,
      "intervals": []
    }
  ]
}
```

### Проверка состояния приложения и базы данных

`GET /health`

Пример ответа:

```json
{
  "status": "ok",
  "database": "ok"
}
```
